Facilitation plan · for the presenter

Leading with AI

A working session for school leaders. Parent-workshop structure, leader-level content. We model it, we govern it, we each build one thing. Leaders leave having built, not just heard about it.

40 to 45 minutes40 to 45 leadersOne device eachTrios for the build

The big idea

You cannot govern what you have only been briefed on. A brief tells you what a tool did once; building one shows you how it breaks, where it drifts, and what you must never let it do. So the room builds.

The through-line. Every beat returns to one posture: we PILOT it and stay in command. Plan, Instruct, Launch, Oversee, Tune. The machine pair (Co-Author, Govern) serves the human pair (Transform, Stay Human).

Three questions, one level up

The national frame names what students do: learn about, use, learn with, learn beyond AI. Ours names what leaders do.

Use it

Model the loop and the agent in the open. Show, do not tell.

Govern it

Decide what data goes where. Set postures per task, by data and stakes.

Keep a school that learns

Guard the cultural and human forces no model reaches.

The set-up

Two screens, one room

Presenter_SL.html on the main screen: you drive it. Audience_SL.html on each leader's device: their screen follows yours, live.

They join with the room code shown on the welcome slide. Default is S3SL. Add ?room=S3SL to either URL to set it directly. Hyphens and spaces in the code do not matter: S3-SL, S3SL, and s3 sl all join the same room.

Grouping

Seat in trios from the start: enough for the build's three roles, Maker, User, Governor. With 40 to 45 leaders that is 13 to 15 trios.

Mixed experience per trio works best. The confident leader becomes the Maker; the cautious one makes a sharp Governor.

The sync uses an MQTT broker over WebSocket, so leaders can join from any network, including their own mobile data; they do not all need the venue wifi. Still, test a real join on the day, before leaders arrive, to confirm the broker is reachable from the room.

The hands-on build · run sheet

Twelve minutes. Three roles in each trio. Each person sends only a handful of messages. Keep agents, loops, and connectors on your account; the rest runs on free accounts.

A · Co-Author
Maker · 8 min
I'm building a reusable writing skill. Ask me six questions, one at a time, to capture how I write parent letters: tone, things I always do, things I never do, length, structure, and one short example.

Answer the six, then:

Now write it as a SKILL.md: a name, a "when to use" line, my voice rules as bullets, the format, and my example. Keep it tight. Inform only.

Copy the SKILL.md. Hand it to the User.

B · Co-Author
User · 3 min

Paste the Maker's SKILL.md, then:

Use this skill. Draft a parent letter about the science fair on 14 March. Inform only, minimal words.

Change one fact and watch it hold the voice:

Update: a wet-weather plan has been added, indoors if it rains. Redo.
C · Govern
Governor · ongoing

Paste the draft, then:

What in this is sensitive, and what should never be put into a public AI tool?

Then:

Now argue the strongest case against sending this at all, in five bullets.
The point. Maker instructs. User launches. Governor oversees. That is PILOT, in twelve minutes. A skill made by one person works for another: capture judgement once, reuse it. Ship something a colleague could use unaided. We fix in the open.

Ship it · the GitHub steps

The deck carries these five steps on the Ship it slide. Say them slowly; they are the take-home capability.

  1. Ask Claude for one file. "Give me this as a single index.html I can host." Download it.
  2. Make a GitHub account. github.com, Sign up. Free.
  3. New repository. Name it, e.g. family-faq. Set it Public. Create.
  4. Upload and switch on. Add file, Upload, drop the index.html, Commit. Then Settings, Pages, Branch: main, Save.
  5. Share the link. Live in about a minute at your-name.github.io/family-faq.
The guardrail, again. A public page is public data. Nothing sensitive goes in: no student or staff records, no internal documents, nothing official. The data line holds here too.

Tensions to hold, not solve

Name these in the room. They are leadership questions, not technical ones.

The polarities

  • Efficiency and effort
  • Speed and depth
  • Trust and verify
  • Standardise and keep the human voice

The ethical load

  • Privacy: monitoring versus trust
  • Bias: only as fair as the data
  • Opacity: black boxes
  • Vendor: their incentives, not yours
  • Autonomy: quietly eroded

MOE AIEd holds the line on four principles: agency, inclusivity, fairness, safety. The human decides.

Guardrails to repeat

  • The data line. Free Claude is for open work only. Student and staff records stay on the government platforms. Decide what data goes where before you build.
  • The maker's mirror. A skill carries one person's voice and blind spots. Build shared skills with many hands, and review what they quietly assume.
  • The category error. The cultural and human forces are not optional. AI clears the admin so you have more hours for people, not fewer.
  • The posture, not the ladder. Partner does not beat Prohibit. The right posture is the one the task calls for.

Before leaders arrive

  • Open Presenter_SL.html on the main screen. Confirm the room code reads S3SL (or set your own with ?room=).
  • On a spare device, open Audience_SL.html, join the room, and confirm the connected count climbs and the screen follows when you press Next.
  • Test a real join from a phone on mobile data, not only the venue wifi. If the connected count does not climb, the broker is blocked on that network; fall back to the printed cards.
  • Have your two paid accounts ready for the agent, connector, and loop demos in See it and Watch it run.
  • Pre-load one worked house-voice SKILL.md so the build has a fallback if a Maker stalls.
  • If you will show shipping live, have a throwaway GitHub account signed in and a spare index.html ready to upload.
  • Seat the room in trios. Put up the room code before the doors open.
  • Keep leadai.html open in a tab: the deep reference each leader takes home.
